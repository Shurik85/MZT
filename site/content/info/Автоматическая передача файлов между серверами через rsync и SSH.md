# 🔄 Инструкция по автоматической передаче файлов с одного сервера на другой через `rsync` и SSH

## 🎯 Цель

Организовать автоматическую синхронизацию данных:

- с **сервера-источника** `111.111.111.111` (SSH-порт `22077`)
- на **резервный сервер** `222.222.222.222` (SSH-порт `33088`)
- с использованием пользователя `user_backup` на резервном сервере.

Синхронизация выполняется для одной из директорий:

- `/opt`
- `/srv/local_backup`

---

## 📌 Условия

- 🖥️ **Сервер 1 (источник):** `111.111.111.111`, SSH-порт `22077`
- 🗄️ **Сервер 2 (резервный):** `222.222.222.222`, SSH-порт `33088`
- 👤 На резервном сервере создаётся пользователь `user_backup`

---

## 1️⃣ Создание пользователя `user_backup` на сервере `222.222.222.222`

1. Подключитесь к резервному серверу как `root`:

```bash
ssh -p 33088 root@222.222.222.222
```

2. Создайте пользователя:

```bash
adduser user_backup
```

3. Добавьте его в группу `sudo`:

```bash
usermod -aG sudo user_backup
```

4. Проверьте, что `sudo` работает:

```bash
su - user_backup
sudo ls /root
```

---

## 2️⃣ Настройка SSH-ключей для передачи без пароля

Все команды ниже выполняются на сервере-источнике `111.111.111.111`.

1. Сгенерируйте SSH-ключ:

```bash
ssh-keygen -t rsa -b 4096 -C "backup@111.111.111.111" -f ~/.ssh/id_rsa
```

2. Передайте публичный ключ на резервный сервер:

```bash
ssh-copy-id -i ~/.ssh/id_rsa.pub -p 33088 user_backup@222.222.222.222
```

Если `ssh-copy-id` недоступен, используйте ручной вариант:

```bash
cat ~/.ssh/id_rsa.pub | ssh -p 33088 user_backup@222.222.222.222 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```

3. Проверьте вход без пароля:

```bash
ssh -p 33088 user_backup@222.222.222.222
```

---

## 3️⃣ Синхронизация данных через `rsync`

### 🗂️ Вариант 1: синхронизация `/opt` → `/srv/local_backup`

```bash
rsync -avz -e "ssh -p 33088" /opt/ user_backup@222.222.222.222:/srv/local_backup/
```

### 🗂️ Вариант 2: синхронизация `/srv/local_backup` → `/srv/local_backup`

```bash
rsync -avz -e "ssh -p 33088" /srv/local_backup/ user_backup@222.222.222.222:/srv/local_backup/
```

### ℹ️ Пояснение ключей

- `-a` — архивный режим (права, время, симлинки и т.д.)
- `-v` — подробный вывод
- `-z` — сжатие при передаче
- `-e "ssh -p 33088"` — использование SSH с нестандартным портом резервного сервера

> ⚠️ В `rsync` указывается порт **сервера назначения**, к которому выполняется подключение. В этом кейсе это `33088`.

---

## 4️⃣ Автоматизация через `cron`

На сервере `111.111.111.111` откройте crontab:

```bash
crontab -e
```

Добавьте задачу ежедневной синхронизации в 03:00:

```bash
0 3 * * * rsync -avz -e "ssh -p 33088" /opt/ user_backup@222.222.222.222:/srv/local_backup/
```

---

## ✅ Заключение

После выполнения шагов вы получите автоматическую и безопасную синхронизацию файлов между серверами по SSH-ключу, без ручного ввода пароля.
