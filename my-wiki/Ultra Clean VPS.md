# 🧹 Ultra Clean VPS (Ubuntu 24.04)

Скрипт для **глубокой очистки VPS на Ubuntu 24.04**.

Подходит для серверов:

* 💾 с маленьким диском (10–20GB)
* 🐳 Docker серверов
* ⚡ VPS после долгой работы
* 🚀 новых серверов перед развёртыванием сервисов

Скрипт удаляет:

* ненужные пакеты
* кеши APT
* старые ядра
* journald логи
* snap
* firmware
* лишние локали
* Docker мусор

Это позволяет **освободить от 1GB до 6GB места на диске**.

---

# ⚡ Быстрая установка (одной командой)

Самый простой способ запустить скрипт:

```bash
curl -fsSL https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/Ultra_Clean_VPS.sh | sudo bash
```

Скрипт автоматически выполнит очистку системы.

# 💾 Бэкап перед очисткой (рекомендуется)

Если на диске есть запас места, перед запуском очистки можно сохранить удаляемые данные в архив.

### Вариант одной командой (бэкап в `/backup_ultra_clean`)

```bash
sudo bash -c 'command -v tar >/dev/null || (apt update && apt install -y tar); TS=$(date +%F_%H-%M-%S); mkdir -p /backup_ultra_clean; tar -czf /backup_ultra_clean/ultra_clean_backup_${TS}.tar.gz /var/log/journal /var/lib/apt/lists /var/cache/apt/archives /tmp /var/tmp /var/lib/docker 2>/dev/null || true; ls -lh /backup_ultra_clean'
```

Что делает команда:

* проверяет наличие `tar` (обычно уже установлен в Ubuntu 24.04)
* при необходимости устанавливает архиватор
* создаёт каталог `/backup_ultra_clean`
* собирает архив с датой и временем в имени
* показывает итоговый размер бэкапа.

После этого можно запускать `Ultra_Clean_VPS.sh`.

> ⚠️ Если Docker не установлен или часть путей отсутствует — это нормально, команда продолжит работу.

# 🎯 Что делает скрипт

---

# 1️⃣ Очистка APT

Удаляются:

* кеш пакетов
* ненужные зависимости
* остатки удалённых пакетов

Это освобождает место и очищает систему.

---

# 2️⃣ Очистка системных логов

Linux хранит логи в:

```
/var/log/journal
```

Со временем они могут занимать **сотни мегабайт или даже гигабайты**.

Скрипт ограничивает их:

* максимум **7 дней**
* максимум **100MB**

Это предотвращает разрастание логов.

---

# 3️⃣ Очистка индексов пакетов

Удаляются старые APT списки:

```
/var/lib/apt/lists
```

После чего выполняется:

```
apt update
```

Это освобождает место и обновляет индекс пакетов.

---

# 4️⃣ Очистка Docker

Если Docker установлен, выполняется:

```
docker system prune -a
```

Удаляются:

* unused images
* stopped containers
* build cache
* unused networks

Это одна из **самых частых причин заполнения диска на VPS**.

---

# 5️⃣ Ограничение Docker логов

По умолчанию Docker пишет **бесконечные логи**.

Скрипт устанавливает лимиты:

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

Теперь один контейнер может хранить максимум:

```
≈ 30MB логов
```

Это предотвращает заполнение диска.

---

# 6️⃣ Удаление документации

Удаляются:

```
/usr/share/doc
/usr/share/man
```

На VPS они почти никогда не используются.

Экономия:

```
100-300MB
```

---

# 7️⃣ Удаление Snap

Ubuntu часто устанавливает:

```
snapd
```

Но на большинстве VPS он:

* не используется
* занимает 200-400MB
* создаёт loop устройства

Скрипт полностью удаляет:

* snap
* snapd
* snap cache

---

# 8️⃣ Очистка локалей

Linux хранит локали для **десятков языков**.

Это может занимать **200-500MB**.

Скрипт оставляет только:

```
en_US.UTF-8
en_GB.UTF-8
ru_RU.UTF-8
```

Остальные удаляются.

---

# 9️⃣ Удаление firmware

Пакет:

```
linux-firmware
```

занимает **300-600MB**.

На VPS он обычно не нужен, потому что:

* VPS работает на виртуальном железе
* реальные драйверы устройств не используются

Поэтому пакет удаляется.

⚠️ На **bare-metal серверах** лучше оставить.

---

# 🔟 Удаление toolchains

Иногда на сервере остаются:

```
llvm
clang
компиляторы
```

Если сервер не используется для сборки программ — они не нужны.

Скрипт удаляет:

```
llvm-*
```

---

# 1️⃣1️⃣ Очистка старых ядер

Ubuntu хранит несколько версий ядра.

Это может занимать **сотни мегабайт**.

Скрипт оставляет:

* текущее ядро
* одно последнее

Остальные удаляются.

Это безопасная практика.

---

# 1️⃣2️⃣ Очистка временных файлов

Удаляются временные файлы из:

```
/tmp
/var/tmp
```

Это освобождает место и удаляет мусор.

---

# 1️⃣3️⃣ Финальный отчёт

После завершения скрипт показывает:

### использование диска

```
df -h
```

### самые большие директории

```
du -xh /
```

### использование Docker

```
docker system df
```

Это помогает понять:

* что занимает место
* сколько удалось освободить.
