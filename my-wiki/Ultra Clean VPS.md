# 🧹 Ultra Clean VPS (Ubuntu 24.04)

Скрипт для **глубокой очистки VPS на Ubuntu 24.04**, особенно полезен для:

* 💾 **маленьких дисков (10–20GB)**
* 🐳 **Docker серверов**
* 🚀 **чистых production VPS**
* 📦 **новых инсталляций**

Скрипт удаляет **мусорные пакеты, логи, кеши, старые ядра и ненужные компоненты**, освобождая место и предотвращая дальнейшее разрастание системы.

---

# 🎯 Зачем это нужно

На небольших VPS часто возникают проблемы:

* диск быстро заполняется
* Docker разрастается
* старые ядра остаются в системе
* APT кеши занимают место
* Snap занимает сотни мегабайт
* journald растёт бесконечно
* локали занимают сотни мегабайт

Этот скрипт решает все эти проблемы **одним запуском**.

---

# ⚠️ Важно

Скрипт **безопасен для VPS**, но может быть **нежелателен для обычных десктопов**.

Некоторые шаги:

* удаляют **Snap**
* удаляют **firmware**
* удаляют **документацию**
* удаляют **старые ядра**

На **обычном сервере / VPS это нормально**, но на **desktop машине может быть нежелательно**.

Инструмент можно установить двумя способами:

⚡ быстрая установка одной командой
📥 ручная установка через скачивание скрипта

---

⚡ Быстрая установка (одной командой)

Самый простой способ установить инструмент на новый сервер:

```
curl -fsSL https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/control-docker-v5.2-cli.sh | sudo bash
```

После установки команда будет доступна глобально:

```
control-docker
```

📥 Ручная установка

Если вы хотите скачать скрипт и установить его вручную.

1️⃣ Создать папку проекта

```
sudo mkdir -p /opt/control-docker/log
```

Эта папка будет использоваться для:

хранения скрипта

хранения логов контейнеров

Структура будет выглядеть так:

/opt/control-docker
 ├─ control-docker-v5.2-cli.sh
 └─ log/

2️⃣ Скачать скрипт

```
cd /opt/control-docker
sudo wget https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/control-docker-v5.2-cli.sh
```

Или через curl:

```
sudo curl -O https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/control-docker-v5.2-cli.sh
```

3️⃣ Сделать скрипт исполняемым

```
sudo chmod +x /opt/control-docker/control-docker-v5.2-cli.sh
```

4️⃣ Создать глобальную команду

Чтобы запускать инструмент одной командой:

```
sudo ln -sf /opt/control-docker/control-docker-v5.2-cli.sh /usr/local/bin/control-docker
```

5️⃣ Обновить кеш команд

```
hash -r
```

❓ Зачем это нужно

Bash кеширует пути к командам.

После создания новой команды control-docker необходимо очистить этот кеш, чтобы оболочка увидела новую команду.

▶️ Запуск

После установки инструмент можно запустить командой:

```
control-docker
```

Откроется интерактивное CLI меню управления Docker.

# 🔧 Что делает скрипт

---

# 1️⃣ Очистка APT

```bash
apt clean
apt autoclean
apt autoremove --purge
```

Удаляет:

* старые пакеты
* кеш загрузок
* зависимости которые больше не нужны

Дополнительно очищаются **остатки удалённых пакетов**:

```
rc packages
```

Это пакеты, которые были удалены, но их конфиги остались.

---

# 2️⃣ Очистка journald логов

Linux хранит системные логи в:

```
/var/log/journal
```

Со временем они могут занимать **гигабайты**.

Скрипт ограничивает их:

* максимум **7 дней**
* максимум **100MB**

Это предотвращает разрастание логов.

---

# 3️⃣ Пересоздание APT lists

Удаляются старые индексы пакетов:

```
/var/lib/apt/lists
```

После чего выполняется:

```
apt update
```

Это позволяет:

* освободить место
* обновить индекс пакетов

---

# 4️⃣ Очистка Docker

Если Docker установлен:

```
docker system prune -a
```

Удаляются:

* unused images
* stopped containers
* build cache
* unused networks

Это **частая причина заполнения диска**.

---

# 5️⃣ Ограничение Docker логов

По умолчанию Docker пишет **бесконечные логи**.

Один контейнер может создать **гигабайты логов**.

Скрипт устанавливает лимиты:

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

Теперь:

* лог максимум **10MB**
* хранится **3 файла**

Максимум на контейнер:

```
≈ 30MB
```

---

# 6️⃣ Удаление документации

Linux пакеты устанавливают:

```
/usr/share/doc
/usr/share/man
```

На VPS эти файлы обычно **никогда не используются**.

Удаление освобождает **100-300MB**.

---

# 7️⃣ Удаление Snap

Ubuntu часто устанавливает:

```
snapd
```

Но на VPS он обычно **не нужен**.

Snap:

* занимает **200-400MB**
* создаёт фоновые сервисы
* создаёт loop devices

Скрипт полностью удаляет:

```
snap
snapd
snap cache
```

---

# 8️⃣ Очистка локалей

Linux хранит **локали для десятков языков**.

Это может занимать **200-500MB**.

Скрипт оставляет только:

```
en_US
en_GB
ru_RU
```

Все остальные удаляются.

Это значительно уменьшает размер системы.

---

# 9️⃣ Удаление firmware

Пакет:

```
linux-firmware
```

занимает **300-600MB**.

На VPS он **обычно не нужен**, потому что:

* VPS работает на виртуальном железе
* драйверы устройств не используются

Поэтому пакет удаляется.

⚠️ Если сервер **bare metal**, лучше оставить.

---

# 🔟 Удаление toolchains

Иногда на сервере остаются:

```
llvm
clang
компиляторы
```

Если сервер не используется для сборки ПО — они не нужны.

Скрипт удаляет:

```
llvm-*
```

---

# 1️⃣1️⃣ Очистка старых ядер

Ubuntu сохраняет старые ядра.

Это может занимать **сотни мегабайт**.

Скрипт оставляет:

* текущее ядро
* одно последнее

Все остальные удаляются.

Это безопасная практика.

---

# 1️⃣2️⃣ Очистка временных файлов

Удаляются временные файлы из:

```
/tmp
/var/tmp
```

Они могут накапливаться со временем.

---

# 1️⃣3️⃣ Финальный отчёт

В конце скрипт показывает:

### Использование диска

```
df -h
```

### Самые большие папки

```
du -xh /
```

### Использование Docker

```
docker system df
```

Это помогает понять **что занимает место**.
