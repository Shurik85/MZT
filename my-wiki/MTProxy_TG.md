# 🚀 MTProxy для Telegram + Каскадная схема (RU → EU)

📡 **Эстония указана как пример.**  
Подойдёт **любой сервер за пределами России**.

Для наглядности используем такие IP:

| Сервер | IP |
|------|------|
| 🇪🇪 Эстония (основной MTProxy) | `11.111.111.11` |
| 🇷🇺 Москва (каскад) | `22.222.222.22` |

---

# ⚡ Установка и запуск MTProxy (в один клик)

Скрипт автоматически:

- устанавливает **MTProxy**
- открывает порт
- генерирует **secret**
- выводит **готовую ссылку подключения**

Запуск одной командой:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/tg_mtproxy.sh)
````

После запуска появится **интерактивное меню** управления прокси.

---

# 📥 Альтернативный способ (скачать скрипт)

wget https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/tg_mtproxy.sh

# 🌍 Поднимаем MTProxy на зарубежном сервере

📍 В нашем примере это сервер **Эстонии**

IP сервера:

```
11.111.111.11
```

Рекомендуемые порты:

| Порт      | Когда использовать    |
| --------- | --------------------- |
| **443**   | стандартный вариант   |
| **1194**  | если 443 занят VPN    |
| **51820** | если 443 занят сайтом |

Например используем **1194**

# ✅ Проверяем прямое подключение

1️⃣ Отправляем ссылку себе в **Telegram**
2️⃣ Нажимаем на неё
3️⃣ Telegram автоматически предложит **добавить прокси**

Если прокси работает стабильно —
🎉 **на этом можно остановиться и пользоваться им.**

---

# 🔀 Настройка каскадного MTProxy

Каскад полезен если:

* 📍 есть сервер **в России**
* 📡 прямой прокси работает нестабильно
* 🚫 требуется обход некоторых блокировок

Схема будет выглядеть так:

```
Telegram
   ↓
🇷🇺 Москва
   ↓
🇪🇪 Эстония
   ↓
Telegram сеть
```

---

# ⚙️ Настройка каскада

На **Московском сервере** запускаем **тот же скрипт**

```bash
bash <(curl -Ls https://raw.githubusercontent.com/r00t-man/MZT/refs/heads/main/files/tg_mtproxy.sh)
```

В меню выбираем:

```
3) Настроить правило MTProxy Kaskad
```

# 💡 Полезные советы

🔹 Используйте **443**, если порт свободен <br>
🔹 Если сервер уже использует **VPN** — используйте `1194` <br>
🔹 Если сервер использует **WireGuard** — используйте `51820` <br>
